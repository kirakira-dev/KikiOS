Hardware Support

QEMU virt (Primary)

  • CPU: Cortex-A72 emulation
  • RAM: 256MB - 4GB (auto-detected)
  • Display: ramfb 800x600
  • Storage: virtio-blk (FAT32)
  • Input: virtio keyboard/mouse
  • Network: virtio-net Ethernet
  • Audio: virtio-sound

Raspberry Pi Zero 2W

  • CPU: BCM2710A1 (Cortex-A53)
  • RAM: 512MB
  • Display: HDMI framebuffer
  • Storage: SD card via EMMC
  • Input: USB keyboard via DWC2
  • No networking or audio on Pi

Boot Process

  1. Bootloader loads at 0x0 (QEMU) or 0x80000 (Pi)
  2. Sets up stack, clears BSS, transitions EL3 to EL1
  3. Kernel parses device tree for RAM
  4. Initializes drivers, mounts FAT32
  5. Spawns /bin/desktop
